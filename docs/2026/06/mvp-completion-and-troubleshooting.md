# MVP完成までの記録 (2026-06-27)

## 状態

要件1〜6（プロジェクト管理・グリッドエディタ・キー役割割り当て・任意形状の結合・フォント取込とラスタライズ・実機への実APK書き出し）を一通り実装し、実機上で「プロジェクト作成→キー編集→結合→Export→インストール→実際に入力できる」という一連の流れを確認できた。アーキテクチャの詳細は同月の[keyboard-factory-architecture.md](keyboard-factory-architecture.md)を参照。本ドキュメントはMVPに到達するまでの実機試行錯誤の記録。

## 実機テストで見つかったバグと根本原因

### 1. 結合キーの表示が崩れる／splitで直らない

- **症状**: `a b / b b`のようなL字結合をすると、aのセルが消えて2×2の塗りつぶしに見える。splitしても直らない。
- **原因1（描画）**: `KeyboardGridView`が結合セルごとに`Path.addRect`を個別に呼んでいたため、結合内部の境界線が消えずに残っていた。
- **原因2（データ）**: `ShapeNormalizer.unmerge`が、分離対象が結合グループの本体（左上セル）だった場合に何もしない実装だった。本体セルが別の結合に巻き込まれて文字が上書きされると、本体だからという理由でsplitが効かず詰む。
- **修正**: 描画は`Region`の論理和→`getBoundaryPath`で外周のみ描画するように変更。`unmerge`は本体セルでも分離できるよう一般化（残ったメンバーの中から新たな本体を再選出）。

### 2. L字／階段状結合でラベル・画像が見切れる

- **原因**: ラベル/画像の配置を結合形状の「外接矩形の中心」で計算していたため、L字の場合は外接矩形の中心が形状の外（窪み部分）に来てしまう。
- **修正**: 結合セルのうち最も左上の1セルを基準に配置するよう変更（データモデルは変更不要、描画時に`ownedCells`から都度算出）。

### 3. 結合ダイアログの右ボタンが見えない

- **原因**: 上下左右4つの結合ボタンを1行に並べていたため、ダイアログ幅から右ボタンが押し出されていた（実在するが見切れていた）。
- **修正**: 上下／左右で2行に分割。

### 4. インストール時の解析エラー（日本語名のプロジェクト）

- **原因**: package名生成でKotlinの`isLetterOrDigit()`を使っていたため日本語文字もそのまま通過し、`android.keyboard.generated.テスト5`のような無効なpackage名のAPKを生成していた。
- **修正**: プロジェクト名をUTF-8バイト列にしてhex文字列化し、それをpackage名に使う方式に変更（`_`への圧縮だと情報量が失われ、同名衝突的な紛らわしさが増える、というユーザー指摘を反映）。表示名（アプリラベル）自体は元々Unicode対応なので日本語名はそのまま使える。

### 5. 謎の「Generated Keyboard」アプリが勝手にインストールされる

- **原因**: 開発用`build.sh`がルート無指定で`installDebug`を実行していたため、Gradleが`:app`だけでなく内部用テンプレート`:keyboard-template`も独立したアプリとしてインストールしてしまっていた。
- **修正**: `build.sh`を`:app`専用タスクに変更。あわせて、別プロジェクトから持ち込まれて無関係だったASanライブラリ同期処理（`app/src/main/jniLibs/`にバイナリをコピーしてAPKを肥大化させていた）も削除。

### 6. 生成キーボードがフルスクリーン表示になる

- **症状**: 入力自体は機能するが、キーボードが画面下部の通常サイズではなく画面全体に表示される。
- **試したが効かなかった対策**: `KeyboardGridView`に`layoutParams`で固定の高さを設定。`onEvaluateFullscreenMode()`を`false`に。IMEサービスの`<application>`から`Theme.DeviceDefault`を除去。
- **本当の原因**: `InputMethodService`が`onCreateInputView()`の戻り値を内部コンテナに追加する際、子Viewが自分で設定した`LayoutParams`の高さを無視し、常にフルスクリーン相当のMeasureSpecを渡してくる。
- **修正**: `onMeasure`をオーバーライドして渡されたMeasureSpecを無視し、自分で高さを強制する`FixedHeightContainer`（FrameLayoutサブクラス）でラップする方式に変更。この方式は実機でのスクリーンショット確認（後述）で実際に直ったことを確認済み。
- **教訓**: カスタムIMEの入力Viewの高さは、`LayoutParams`をいくら設定しても効かない場合がある。`onMeasure`で強制するのが確実。

### 7. IMEピッカーでのキーボード名が「Generated Keyboard」のまま

- **原因**: テンプレートの`<service>`要素が独自の`android:label="@string/app_name"`を持っていたため、`<application>`側のラベルをARSCLibでパッチしても`<service>`側は元の文字列リソースを参照したままだった。
- **修正**: `<service>`から`android:label`を削除し、`<application>`のラベル（パッチ対象）を継承させるようにした。

## 実機検証の進め方

このマシンには動くAndroidエミュレータがない（`/dev/kvm`にアクセス不可）。実機（USB、WSL側は`usbipd-win`経由）で検証している。接続のたびに`~/adb_connect_usb.sh`の実行が必要（詳細は[CLAUDE.md](../../../CLAUDE.md)）。

バグ6の最終確認では、ユーザーの実機操作を待たずに、Claude側で`adb shell ime set`でIMEを切り替え、`adb shell input tap`でテキスト欄をフォーカスし、`adb exec-out screencap`でスクリーンショットを撮って直接見た目を確認する、という手法が有効だった（検証用に別の署名で一時的にインストールしたため、確認後はアンインストールしてユーザー本来の署名鍵での再インストールができる状態に戻した）。
